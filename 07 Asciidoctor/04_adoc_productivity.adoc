Diese VS Code-Erweiterung hilft dir, Dokumentationen, Entwürfe und Berichte in AsciiDoc deutlich schneller zu schreiben. Sie fügt automatisch Quellcode, Tabellen und Bilder ein. So konzentrierst du dich auf den Inhalt und musst nicht ständig die Syntax beachten.

== Installation

Suche im Erweiterungs-Manager nach dem Paket `asciidoc-productivity`. Alternativ kannst du die Erweiterung direkt im Marketplace öffnen: https://marketplace.visualstudio.com/items?itemName=schletz.asciidoc-productivity.

== Features

Die vollständige Funktionsliste findest du in der Datei link:asciidoc-productivity/README.md[README.md].

=== LLM-Server für AI-Features

Die KI-Funktionen „Translate“ und „Check spelling and grammar“ benötigen einen OpenAI-kompatiblen Endpunkt. Du kannst zum Beispiel Ollama oder vLLM verwenden.

==== Betrieb mit LM Studio

Installiere zuerst link:https://lmstudio.ai/[LM Studio].
Suche im Bereich _Model search_ nach dem Modell *lmstudio-community/gemma-4-E4B-it-GGUF*.
Wähle die Quantisierung _Q8_.
Lade das Modell herunter.
Passe danach die _context length_ auf _8192 Tokens_ an.
Ein Token entspricht ungefähr 3,5 Zeichen.
Der gesamte Kontext (Text und Antwort) umfasst also etwa 28.500 Zeichen.
Das bedeutet: Du kannst mit dieser Einstellung ca. 14.000 Zeichen übersetzen.

image::lm_studio_1620.png[]

Aktiviere danach im Bereich _Developer_ den Server. Der Server stellt dir die benötigten OpenAI-kompatiblen Endpunkte bereit.

Öffne nun die Benutzereinstellungen von VS Code über _Preferences: Open User Settings (JSON)_. Trage dort folgende Einstellungen für die KI-Funktionen ein:

[source,json]
----
"asciidoc-productivity.completionsUrl": "http://127.0.0.1:1234/v1/chat/completions",
"asciidoc-productivity.llm": "lmstudio-community/gemma-4-E4B-it-GGUF",
"asciidoc-productivity.maxOutputTokens": 4096,
----

[IMPORTANT]
Die Antwort endet nach 4.096 Tokens automatisch. Prüfe daher immer, ob der Text vollständig ist.
Für längere Texte kannst du den Wert `maxOutputTokens` auf 8.192 erhöhen.
Passe dann auch die _context length_ in LM Studio auf 16.384 Tokens an. Der Kontext besteht aus System Prompt, Eingabetext und Antwort.

===== Für PCs mit 32 GB VRAM 💰💰💰

Verfügst du über 32 GB VRAM oder nutzt du einen Mac mit Unified Memory, kannst du auch größere Modelle verwenden.
Teste zum Beispiel das Modell link:https://huggingface.co/lmstudio-community/Qwen3.6-35B-A3B-GGUF[lmstudio-community/Qwen3.6-35B-A3B-GGUF] in der Quantisierung Q4_K_M.
Stelle die _Context Length_ auf 65536 ein. So steht genug Platz für das Reasoning zur Verfügung.
Lade das Modell mit folgenden Einstellungen:

[source,json]
----
"asciidoc-productivity.completionsUrl": "http://127.0.0.1:1234/v1/chat/completions",
"asciidoc-productivity.llm": "lmstudio-community/Qwen3.6-35B-A3B-GGUF",
"asciidoc-productivity.maxOutputTokens": 32768,
----

=== Verwendung von Cloud-Services

Wenn du AI-Services in der Cloud nutzen möchtest, kannst du zum Beispiel auf https://aistudio.google.com einen API-Endpunkt erstellen.
Google bietet ein großzügiges Free Tier an.
Dieses begrenzt nicht die Anzahl der Tokens, sondern nur die Rate (Häufigkeit).
Für unsere Zwecke ist dies völlig ausreichend.
Wir müssen keine Dokumente jede Minute übersetzen.
Beachte allerdings, dass deine Daten von Google im Free Tier für das Modelltraining verwendet werden.
Das ist der "Preis" dafür.
Erstelle zunächst einen API-Key und füge ihn dann in den Einstellungen ein:

[source,json]
----
{
  "asciidoc-productivity.completionsUrl": "https://generativelanguage.googleapis.com/v1beta/openai/chat/completions",
  "asciidoc-productivity.llm": "gemini-2.5-flash",
  "asciidoc-productivity.apiKey": "DEIN-ECHTER-GEMINI-API-KEY-HIER",
  "asciidoc-productivity.maxOutputTokens": 32768,
}
----
